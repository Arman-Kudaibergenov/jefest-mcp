Архитектура

Каждая сессия Claude Code работает с 12 PowerShell хуками, настроенными в ~/.claude/settings.json. Строка статуса записывает состояние контекста на каждом ходу. Хуки срабатывают на 7 различных событиях, записывая в RLM (долгосрочная память), локальные JSONL буферы и файлы сигналов диспатча. Вместе они образуют 3-слойную систему защиты: охранники реального времени, сохранение до компакта и восстановление после компакта.

PreToolUse — Охранники

Срабатывают до каждого вызова инструмента. Могут разрешить, отклонить или добавить additionalContext.

context-guard.ps1
~/.claude/hooks/context-guard.ps1
Читает $TEMP/claude-ctx-state.json. Вычисляет remaining = effectiveLimit - tokens. WARN (<50k remaining): allow + предупреждение. CRIT (<30k remaining): deny всех неразрешённых инструментов. Разрешены: Read, Glob, Grep, инструменты RLM, AskUserQuestion, TaskList, TaskGet. Для диспатч-агентов: пишет result-PROJECT.json и signal-PROJECT.json.
может отклонить читает ctx state сигналы диспатча
bash-guard.ps1
~/.claude/hooks/bash-guard.ps1 | matcher: Bash
Обнаруживает шумные test runner-ы (pytest, npm test и т.д.) без редиректа вывода. Разрешает с предупреждением использовать > /tmp/test-out.txt 2>&1. Предотвращает переполнение контекстного окна большим stdout.
только предупреждение только Bash

PostToolUse — Мониторы и логгеры

Срабатывают после каждого вызова инструмента. Matcher .* = все инструменты, или конкретные матчеры. Stdout = встраивается как system-reminder.

mcp-stats.ps1
~/.claude/hooks/mcp-stats.ps1 | matcher: .*
Логирует каждый вызов инструмента в ~/.claude/mcp-stats.jsonl. Категории: mcp, bash, search, other. Авторотация при 10MB. Отслеживает проверку MCP серверов для строки статуса.
логгер каждый вызов
context-monitor.ps1
~/.claude/hooks/context-monitor.ps1 | matcher: .*
WARN (60k remaining): сообщение в stdout один раз за сессию. CRIT (50k remaining): автосохранение в RLM + записывает rlm-saved.flag + запускает rotate-session.ps1 + CRITICAL сообщение в stdout. Срабатывает один раз за порог за сессию (флаг-файлы).
критическое действие запись в RLM rotate-session
auto-capture.ps1
~/.claude/hooks/auto-capture.ps1 | matcher: .*
Записывает пути файлов Edit/Write и важные Bash команды (git, build, test) в autocapture-buffer.jsonl. Буфер читается хуками pre-compact, session-end и periodic-flush.
запись в буфер Edit/Write/Bash
periodic-flush.ps1
~/.claude/hooks/periodic-flush.ps1 | matcher: Edit|Write | async
Каждые 10 вызовов edit/write: читает буфер autocapture, сохраняет список файлов в RLM. Быстрый путь (<10ms) при счётчике < 10. Счётчик на сессию.
запись в RLM async с троттлингом
dispatch-tool-log.ps1
~/.claude/hooks/dispatch-tool-log.ps1 | matcher: .* | async
Логирует вызовы инструментов с деталями в $JEFEST_DISPATCH_LOG. Нулевые затраты для не-диспатч сессий (env var не установлен = мгновенный выход). Также срабатывает при PostToolUseFailure.
только диспатч async

Цикл компакта — Сохранение и восстановление

pre-compact.ps1
~/.claude/hooks/pre-compact.ps1 | PreCompact
Срабатывает до авто-компакта (~168k токенов). Stdout НЕ встраивается. Читает буфер autocapture + транскрипт (последние 200 строк). Сохраняет в RLM: имя проекта, изменённые файлы, команды, сообщения пользователя. Также сохраняет последние PENDING задачи как отдельный факт.
запись в RLM x2 pre-compact
post-compact-restore.ps1
~/.claude/hooks/post-compact-restore.ps1 | SessionStart compact
Срабатывает после компакта. Stdout встраивается в новый контекст. Ищет в RLM: PENDING задачи, PRE-COMPACT автосохранения. Фильтрует по текущему проекту. Выводит контекст восстановления.
чтение RLM x2 stdout встраивается

SessionEnd — Очистка и сохранение

session-end-save.ps1
~/.claude/hooks/session-end-save.ps1 | async
Полное резюме сессии в RLM: файлы, команды, состояние git, счётчик инструментов, сообщения пользователя. Записывает сигнал диспатча для диспатч-сессий ($JEFEST_PROJECT). Сохраняет PENDING задачи. Очищает буфер autocapture (удаляет записи текущей сессии).
запись в RLM x2 сигнал диспатча async
dispatch-post-exit.ps1
Jefest/scripts/dispatch-post-exit.ps1 | async
Хук специфичный для Jefest. Обрабатывает сигналы завершения диспатча. Читает файлы результатов, обновляет состояние диспатча, запускает relay при необходимости.
диспатч async
session-end-hygiene.ps1
~/.claude/hooks/session-end-hygiene.ps1 | async
Запускает rlm-hygiene.py cleanup --execute не чаще одного раза в 24ч. Удаляет устаревшие/просроченные факты RLM. Троттлинг по временной метке.
гигиена async кулдаун 24ч

SubagentStop

subagent-stop-verify.ps1
~/.claude/hooks/subagent-stop-verify.ps1 | async
Записывает резюме результата агента в RLM. Захватывает: тип агента, ID, последнее сообщение (обрезано до 500 символов). Обеспечивает видимость результатов агентов между сессиями.
запись в RLM async

Защита контекста — 3 слоя

СлойХукТриггерПорогДействие
1 Реальное время context-guard PreToolUse <50k remaining: WARN
<30k remaining: CRIT		 WARN: additionalContext
CRIT: deny инструментов записи
1 Реальное время context-monitor PostToolUse <60k remaining: WARN
<50k remaining: CRIT		 WARN: stdout один раз
CRIT: сохранение в RLM + rotate-session
2 До компакта pre-compact PreCompact ~168k Порог авто-компакта Прямое сохранение в RLM (файлы, команды, PENDING)
3 После компакта post-compact-restore SessionStart compact После события compact Чтение RLM, внедрение PENDING + состояния в новый контекст
Пороги (актуальные): context-guard использует effectiveLimit (168k), CRIT при tokens > 138k. context-monitor использует effectiveLimit (168k), CRIT при tokens > 118k (remaining < 50k). Оба теперь используют effectiveLimit — пороги согласованы, мёртвая зона устранена.

statusline.ps1 — пульс системы

Запускается на каждом ходу. Отображает: модель, проект, использование токенов, % до компакта, статус thinking, здоровье MCP серверов (TCP проверка), лимиты API (5ч/7д/доп.). Записывает $TEMP/claude-ctx-state.json с полями { pct, tokens, limit, effectiveLimit } — единый источник истины для всех контекстно-зависимых хуков. Кэширует данные API на 60 секунд во избежание превышения лимитов запросов.
